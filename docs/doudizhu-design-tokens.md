# 斗地主设计 Token（P0 首版）

> 目标：先给 `apps/client` 一套可直接落地的 CSS token。所有颜色、圆角、阴影、字号、间距和卡牌尺寸先走变量，避免后续 UI 打磨时散落硬编码。

## 1. 视觉方向

- **方向名**：喜庆金绿牌桌。
- **关键词**：明亮、轻快、可读、像斗地主、有一点竞技感。
- **参考边界**：可参考欢乐斗地主的绿色牌桌、金色按钮、红黑牌面信息层级；不直接搬竞品原图、原音频或商标化素材。
- **实现方式**：P0 优先 CSS/SVG/组件拼装，减少图片依赖；后续精修再补位图或动效序列帧。

## 6. 按钮状态

| 状态 | 视觉 |
|---|---|
| 主按钮 normal | 金色渐变 `#f5c542 -> #d99a1e`，深色文字，轻投影 |
| 主按钮 hover | 亮度 +6%，投影加强 |
| 主按钮 active | 下压 `translateY(1px)`，使用 `--ddz-gold-600` |
| 主按钮 disabled | 透明度 0.45，去投影，禁用鼠标 |
| 次按钮 normal | 半透明深绿底 + 浅色描边 |
| 危险按钮 normal | 红色渐变 `#d63031 -> #a82020`，白字 |
| 幽灵按钮 normal | 透明底 + 金色文字/描边 |

## 7. P0 动效

| 场景 | 动效规格 |
|---|---|
| 选牌 | 80ms 上移 8px + 金色 glow；取消选择反向回落 |
| 出牌 | 160ms 从手牌区到出牌区的位移 + 轻微缩放 `0.96 -> 1` |
| 叫地主 | 地主标识 220ms 弹入，按钮区做一次金色闪光 |
| 倒计时 | 10s 内普通环形；3s 内红色脉冲，但尊重 reduced motion |
| 结算胜负 | 结算弹窗 180ms 上浮淡入；胜利用金光，失败用暗红遮罩 |
| 炸弹/王炸 | P0 只做 CSS 爆闪 + 震动 120ms；P1 再补序列帧/Lottie |

## 8. 接入优先级

1. **先接 token**：把 `:root` 现有 `--felt/--gold/...` 映射到 `--ddz-*`，旧变量可暂时兼容。
2. **再接卡牌组件**：`CardView.tsx` 保持组件拼装，优先补牌面、背面、选中态和大小王样式。
3. **再接桌面/座位**：`GameTable.tsx` 使用台呢纹理、座位徽章、地主标识、倒计时和倍数展示。
4. **最后接大厅/房间/结算**：先保证主链路清楚，再做氛围图和弹窗精修。
