# 斗地主设计 Token（P0 首版 · LRM-166 校准）

> 目标：先给 `apps/client` 一套可直接落地的 CSS token。所有颜色、圆角、阴影、字号、间距和卡牌尺寸先走变量，避免后续 UI 打磨时散落硬编码。
>
> **LRM-166（2026-07-21）**：锁定牌桌桌沿 rail、台呢降饱和、出牌态信息层级与卡牌尺寸，供小林按变量接入，不再散落硬编码。

## 1. 视觉方向

- **方向名**：喜庆金绿牌桌。
- **关键词**：明亮、轻快、可读、像斗地主、有一点竞技感。
- **参考边界**：可参考欢乐斗地主的绿色牌桌、金色按钮、红黑牌面信息层级；不直接搬竞品原图、原音频或商标化素材。
- **实现方式**：P0 优先 CSS/SVG/组件拼装，减少图片依赖；后续精修再补位图或动效序列帧。
- **出牌态信息层级（状态 3）**：全屏只允许**出牌**主按钮最亮；倒计时与倍数合并进 `.meta-corner`（弱权重）；身份/剩余牌数常驻但不抢主 CTA。

## 6. 按钮状态

| 状态 | 视觉 |
|---|---|
| 主按钮 normal | 金色渐变 `#f5c542 -> #d99a1e`，深色文字，轻投影；**出牌态唯一满亮度** |
| 主按钮 hover | 亮度 +6%，投影加强 |
| 主按钮 active | 下压 `translateY(1px)`，使用 `--ddz-gold-600` |
| 主按钮 disabled | 透明度 0.45，去投影，禁用鼠标 |
| 次按钮 normal | 半透明深绿底 + 浅色描边；相对主按钮明显降饱和 |
| 危险按钮 normal | 红色渐变 `#d63031 -> #a82020`，白字 |
| 幽灵按钮 normal | 透明底 + 奶油色文字/描边（非金色满亮） |

## 7. 出牌态信息层级（LRM-166）

| 层级 | 元素 | 规格 |
|---|---|---|
| 主 | 「出牌」主按钮 | 唯一使用满亮度 `--ddz-gold-500` |
| 主内容 | 中央出牌区 + 我方手牌 | 正常纸面白，无金色外发光 |
| 次 | `.meta-corner`：倒计时 + 倍数同胶囊 | `opacity ≈ 0.55–0.65`，字号 `xs`；禁止与主按钮同级金色 |
| 背景 | 身份徽章、剩余牌数、座位名 | 常驻弱权重；身份只徽章不叠文字 |

## 8. P0 动效

| 场景 | 动效规格 |
|---|---|
| 选牌 | 80ms 上移 8px + 收敛金色描边；取消选择反向回落 |
| 出牌 | 160ms 从手牌区到出牌区的位移 + 轻微缩放 `0.96 -> 1` |
| 叫地主 | 地主标识 220ms 弹入；**禁止**按钮区大面积金色闪光 |
| 倒计时 | meta-corner 内环形；3s 内红色脉冲，尊重 reduced motion |
| 结算胜负 | 结算弹窗 180ms 上浮淡入；胜利用金光，失败用暗红遮罩 |
| 炸弹/王炸 | P0 只做 CSS 爆闪 + 震动 120ms；P1 再补序列帧/Lottie |

## 9. 接入优先级（小林）

1. **`:root` 对齐本文**：felt / rail / glow / card 尺寸以本文为准（LRM-166）。
2. **`.table` 桌沿**：`box-shadow` 外圈改用 `--ddz-rail-*`；去掉金边「第二桌沿」与大面积 inset 金光。
3. **台呢渐变**：中心/边缘只用 `--ddz-felt-*`，禁止硬编码高饱和绿。
4. **meta-corner**：倒计时 + 倍数同容器；`turn-timer` 尺寸走 `--ddz-timer-size`，常态不加金光晕。
5. **主 CTA**：仅 `.btn.primary`（出牌）满亮；不出/提示降饱和。
