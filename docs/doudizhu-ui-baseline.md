# 斗地主 UI 基准规格（对标腾讯「欢乐斗地主」上线产品）

> 全群唯一验收基准。v7 起所有 UI 审核**只对照本文逐条 diff**，不达标一律打回。
> 起草：小雅。核稿/定稿：贝克汉姆。基准来源：腾讯欢乐斗地主上线产品（对标体验/质量/手感，**不抄其美术资产**，图标/视觉走开源可商用）。
>
> **能力说明**：小雅当前读不了 PNG，无法逐像素看欢乐斗地主截图。本 v1 基于「欢乐斗地主公开通用观感 + caozs2 口径 + Frank 实测反馈」整理；标 ⚠️ 的条目需 caozs2 或能看图的同学按真实截图校准后再锁。校准请直接在对应行后追加 `(校准：...)`。
>
> 每条结构：**维度 | 基准目标(欢乐斗地主) | 现状(main) | 差距/动作 | 校准**

通用底线（所有状态适用）：
- 一屏只允许**一个**最亮的主操作（金色主按钮），其余信息降饱和收进角落。
- 同一信息**只出现一次**（身份不要徽章+文字双叠；玩家名/座位号/身份不要堆叠）。
- 颜色与尺寸**走 `--ddz-*` 设计 token**，不硬编码。
- 图标走开源可商用库（Iconify 等），记录来源+license，不抄竞品原图。
- 对着看久不累：台呢饱和度适中、去金色泛光、柔光过渡。
