# UI 资源需求清单（斗地主 · Web 端）

> 负责人：@小林（前端）｜ 目标读者：UI/视觉设计师
> 用途：把客户端**每个界面需要美术资源的地方**一次性标注清楚，设计师到岗后照本文即可产出全部素材。
> 维护方式：随开发进度更新（小林开发到哪、文档更新到哪）。每项给「用途 / 当前占位 / 规格 / 优先级 / 备注」。

## 0. 视觉总基调（参照腾讯《欢乐斗地主》）

- **P0 方向定稿（小雅首版）**：喜庆金绿牌桌。整体明亮、轻快、可读，绿色台呢 + 金色主按钮 + 红黑牌面，保证一眼像斗地主。
- **参考边界**：可参考腾讯欢乐斗地主的节奏、信息层级和氛围；不直接搬竞品原图、原音频、商标化素材。
- **实现策略**：P0 先用 CSS/SVG/组件拼装交付，减少图片依赖；能合法复用的开源/免费商用素材必须记录来源、license、可否商用。
- **优先级**：`P0` = MVP 上线前必须有；`P1` = 锦上添花，可后补。
- **设计稿交付**：移动端优先（竖屏 ≤ 560px）+ 桌面端（≤ 1100px）两套断点；卡片/按钮请给 `@2x / @3x` 或 SVG。
- **资源目录约定**：放 `apps/client/src/assets/<界面>/`，文件名见每节「命名」。
- **设计 token**：见 `docs/doudizhu-design-tokens.md`，客户端接入优先把颜色、圆角、阴影、字号、卡牌尺寸改走变量。
- **P0 视觉预览**：见 `docs/assets/doudizhu-p0-table.svg`，用于小林按牌桌氛围和信息层级接入首版样式。

## 7. P0 客户端接入优先级（给 `apps/client`）

1. **设计 token**：先把 `apps/client/src/styles.css` 的 `:root` 变量映射到 `docs/doudizhu-design-tokens.md`，保持现有变量兼容，避免影响 MVP 联机逻辑。
2. **卡牌组件**：优先改 `CardView.tsx` / `.card` 样式，使用组件拼装完成红黑牌面、大小王、背面、选中态；不先做 54 张位图。
3. **牌桌与座位**：改 `GameTable.tsx` 相关样式，补台呢纹理、出牌区容器、座位徽章、地主标识、倒计时、倍数展示。
4. **全局按钮**：统一 `.btn.primary/.ghost/:disabled` 的金色、红色、灰色状态，保证叫地主/出牌/结算按钮一致。
5. **大厅/房间/结算氛围**：最后补大厅背景、房间座位卡和结算弹窗，不阻塞已跑通的联网闭环。

## 9. 资源命名与接入约定

- 目录：`apps/client/src/assets/<界面>/<名称>.<ext>`
- 卡牌：若用单图，`assets/cards/<rank>-<suit>.png`（如 `11-heart.png`）；大小王 `joker-small.png` / `joker-big.png`。若用拼装，仅给花色 + 点数素材。
- 矢量优先：Logo、图标、特效用 SVG/Lottie；卡牌给位图（@2x）。
- 接入：设计师把成品替换进 `src/assets/`，小林负责在组件里接上；改了文件名请在群里同步。
- 风格基线：一律向腾讯《欢乐斗地主》靠拢（caozs2 2026-07-15 决定）。
