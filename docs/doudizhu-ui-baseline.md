# 斗地主 UI 基准规格（对标腾讯「欢乐斗地主」上线产品）

> 全群唯一验收基准。v7 起所有 UI 审核**只对照本文逐条 diff**，不达标一律打回。
> 起草：小雅。核稿/定稿：贝克汉姆。基准来源：腾讯欢乐斗地主上线产品（对标体验/质量/手感，**不抄其美术资产**，图标/视觉走开源可商用）。
>
> **能力说明**：小雅当前读不了 PNG，无法逐像素看欢乐斗地主截图。本 v1 基于「欢乐斗地主公开通用观感 + caozs2 口径 + Frank 实测反馈」整理；标 `[需澄清]` 的条目为后续校准项，不阻塞 v1 作为当前唯一 UI diff 源；每项已标明负责人。校准请直接在对应行后追加 `(校准：...)`。
>
> 每条结构：**维度 | 基准目标(欢乐斗地主) | 现状(main) | 差距/动作 | 校准**

通用底线（所有状态适用）：
- 一屏只允许**一个**最亮的主操作（金色主按钮），其余信息降饱和收进角落。
- 同一信息**只出现一次**（身份不要徽章+文字双叠；玩家名/座位号/身份不要堆叠）。
- 颜色与尺寸**走 `--ddz-*` 设计 token**，不硬编码。
- 图标走开源可商用库（Iconify 等）或内部自绘，记录来源+license，不抄竞品原图。
- 对着看久不累：台呢饱和度适中、去金色泛光、柔光过渡。

## 当前 v7 立即生效硬指标

以下是已确认规格门，不属于 `[需澄清]`，LRM-131/LRM-140 后续交付必须立即逐条满足：

| 硬指标 | 立即生效验收口径 |
|---|---|
| 手牌正排 | 我方手牌 `rotate(0deg)`，不得扇形倾斜；所有未选中牌等高，仅允许横向轻微重叠；选中态只做上移，不改变整排水平基线。 |
| 卡牌缩小 | 主手牌目标 `52×74px`；桌面/对手小牌目标 `40×56px`；移动端可等比缩放但不得回到 `64×90px` 的大牌密度。 |
| 身份只留徽章去文字 | 地主/农民身份只保留头像级 SVG/图标徽章；移除“地主/农民”文字牌、重复身份文案和徽章+文字双叠。可保留无视觉重复的 `aria-label`。 |
| 唯一主按钮最亮 | 当前可执行的主动作（叫分/出牌/再来一局）是全屏唯一最高亮金色按钮；不出/提示/返回大厅等次按钮必须降饱和、降亮度。 |
| 台呢降饱和去泛光 | 牌桌台呢绿降饱和、降低明暗跳变；去掉大面积金色外发光/刺眼泛光，只保留柔和边缘过渡和局部高光。 |

## 图形资产门禁（LRM-140）

前端视觉不能只靠 CSS 占位；扑克牌、大小王、牌背、地主/农民身份、加倍/炸弹/春天/结算等关键状态必须有可落地 SVG/图片/组件资产。当前资产规格与路径见 `docs/front-end-visual-assets.md`，交付给小林接入时必须逐项标明：路径、尺寸、用途、状态、版权/来源。

- 扑克牌：`docs/assets/cards/card-front-template.svg`、`card-back.svg`、`joker-small.svg`、`joker-big.svg`。
